# FT_PRINTF

## Description
FT_PRINTF is a custom implementation of the standard C `printf` function. The goal of this project is to understand and recreate the core functionality of `printf`, handling various format specifiers, flags, and data types, while managing memory and output efficiently. This project is a common exercise in the 42 curriculum to deepen understanding of variadic functions, string manipulation, and low-level I/O in C.

## Instructions

### Compilation
To compile the project, use the provided `Makefile`:

```sh
make
```

This will generate the `libftprintf.a` static library. To clean object files, use:

```sh
make clean
```

To remove all generated files (including the library):

```sh
make fclean
```

### Usage
Include `ft_printf.h` in your project and link against `libftprintf.a`:

```c
#include "ft_printf.h"
```

Compile your program with:

```sh
cc -Wall -Wextra -Werror your_program.c libftprintf.a
```

## Algorithm and Data Structure Justification
The core of `ft_printf` relies on parsing the format string and handling variadic arguments. The algorithm iterates through the format string, identifies format specifiers, and dispatches the appropriate handler for each type (character, string, integer, hexadecimal, pointer, etc.).

- **Parsing**: A simple state machine is used to process the format string, switching between normal output and format parsing modes.
- **Data Structures**: No complex data structures are required; the implementation primarily uses arrays to manage state. Variadic arguments are handled using `stdarg.h` macros (`va_start`, `va_arg`, `va_end`).
- **Justification**: This approach is efficient for the requirements of `ft_printf`, as it minimizes overhead and closely mirrors the behavior of the standard library implementation.
